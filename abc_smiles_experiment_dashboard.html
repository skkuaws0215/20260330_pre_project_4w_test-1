1) 실험 개요

같은 split(seed=42, test_size=0.2)에서 A/B/C를 비교. C 재실험은 use_smiles=on으로 SMILES를 문자 n-gram 해시(2048) 피처로 추가.

주석 보기: A/B/C 실험군 정의
  • A (엄격 baseline): 결측치 30% 기준으로 정제된 피처셋. 안정성/보수적 기준 비교군.
  • B (느슨 baseline): 결측치 70% 기준 피처셋 + 문자열 SMILES 제외. 기존 파일럿 기준선.
  • C old: 결측치 70% 원본 기반이지만 기존 학습 코드가 numeric-only라 A/B와 사실상 동일 경로.
  • C new (SMILES on): C 데이터에서 drug__smiles를 해시 피처로 수치화해 기존 numeric 피처에 결합.
주석 보기: FE 생성 기준(무엇을 기준으로 만들었는지)
  • 공통 키: sample_id + canonical_drug_id 기준으로 labels/features를 inner join.
  • 결측 기준: A는 30% 임계, B/C는 70% 임계로 피처 보존 폭을 다르게 설계.
  • 학습 입력 기본: numeric dtype만 사용, 결측은 0.0으로 대체.
  • SMILES 확장(new-C): char n-gram(2~4) + hashing(2048)으로 고정 차원 sparse 벡터 생성.
  • 실행 안정성 기준: 팀 버킷 원본을 SageMaker 버킷으로 AES256 스테이징 후 학습(권한/KMS 실패 재발 방지).

주석 보기: C new에서 RF만 멈춘 이유 (운영 판단)
  • 동일 인스턴스·동일 데이터에서 LGBM/XGB/EN은 수십 분대에 끝났고, RF만 수 시간 단위로 늘어짐.
  • RandomForest는 고차원 sparse 입력에서 트리 분할 비용이 커지는 경우가 많아, 파일럿 4모델 비교 목적과 맞지 않음.
  • 따라서 성능 지표는 3모델로 확정하고, RF는 재시도 없이 종료. 향후 필요 시 n_estimators·max_depth 경량화 또는 RF 제외 루틴으로 정리하는 것이 합리적.

주석 보기: 표의 각 컬럼이 의미하는 것
  • Model: 비교 대상 알고리즘. 같은 split에서 서로 성능 비교.
  • Baseline RMSE: 기존 C old(실질 numeric-only) 검증 RMSE. 낮을수록 오차가 작음.
  • New-C RMSE: SMILES on(C new) 검증 RMSE. baseline보다 낮아지면 개선.
  • Delta (RMSE): New-C - Baseline. 음수면 개선, 양수면 악화.
  • Baseline/New-C Spearman: 예측값-정답의 순위 일치도(단조 관계). 높을수록 순위가 잘 맞음.
  • Delta (Spearman): New-C - Baseline. 양수면 개선.
주석 보기: 숫자값 해석 가이드 (특히 Spearman)
  • RMSE/MAE는 절대오차 크기 지표라서 작을수록 좋다. 단위는 타깃(label_regression) 단위와 동일.
  • Spearman은 “정답 순위와 예측 순위가 얼마나 같은 방향으로 움직이는지”를 본다.
  • 값 범위는 -1 ~ 1:
    • +1: 순위 완전 일치 (가장 이상적)
    • 0 부근: 순위 관계 거의 없음
    • -1: 순위 완전 반대
  • 실무 해석(경험적 가이드):
    • 0.00 ~ 0.10: 거의 무관
    • 0.10 ~ 0.30: 약한 상관
    • 0.30 ~ 0.50: 중간 상관
    • 0.50 ~ 0.70: 강한 상관
    • 0.70+: 매우 강한 상관
  • 왜 Spearman이 중요한가? 약물 스크리닝/랭킹 문제에서는 “절대값 예측 오차”뿐 아니라 우선순위를 맞추는 능력이 핵심이기 때문. Spearman이 높으면 Top 후보 선별의 신뢰도가 올라간다.
  • 이번 결과에서는 baseline 약 0.25(약한 상관)에서 new-C 약 0.48(중간 상관)로 올라, 랭킹 품질이 의미 있게 개선된 것으로 해석 가능.
