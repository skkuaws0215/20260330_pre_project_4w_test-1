  • A (엄격 baseline): 결측치 30% 기준으로 정제된 피처셋. 안정성/보수적 기준 비교군.

주석 보기: FE 생성 기준(무엇을 기준으로 만들었는지)

  • 결측 기준: A는 30% 임계, B/C는 70% 임계로 피처 보존 폭을 다르게 설계.
  • 학습 입력 기본: numeric dtype만 사용, 결측은 0.0으로 대체.
  • SMILES 확장(new-C): char n-gram(2~4) + hashing(2048)으로 고정 차원 sparse 벡터 생성.
  • 실행 안정성 기준: 팀 버킷 원본을 SageMaker 버킷으로 AES256 스테이징 후 학습(권한/KMS 실패 재발 방지).

주석 보기: 표의 각 컬럼이 의미하는 것
  • Model: 비교 대상 알고리즘. 같은 split에서 서로 성능 비교.
  • Baseline RMSE: 기존 C old(실질 numeric-only) 검증 RMSE. 낮을수록 오차가 작음.
  • New-C RMSE: SMILES on(C new) 검증 RMSE. baseline보다 낮아지면 개선.
  • Delta (RMSE): New-C - Baseline. 음수면 개선, 양수면 악화.
  • Baseline/New-C Spearman: 예측값-정답의 순위 일치도(단조 관계). 높을수록 순위가 잘 맞음.
  • Delta (Spearman): New-C - Baseline. 양수면 개선.
주석 보기: 숫자값 해석 가이드 (특히 Spearman)
  • RMSE: 큰 오차에 더 민감(outlier penalty). 모델의 “크게 틀리는 케이스”를 보려는 안정성 지표로 사용.
  • MAE: 평균 절대 오차. 평균적으로 얼마나 틀리는지 보는 실제 예측 정확도 지표.
  • R² (결정계수): 전체 분산 설명력. noisy bio 데이터에서는 높게 안 나올 수 있어 절대 기준보다 상대 비교로 본다.
  • Spearman: 정답 순위와 예측 순위의 상관. drug repurposing에서는 랭킹 품질 때문에 핵심 지표.
  • NDCG@K: 상위 K 순위 품질을 할인 누적 이득으로 평가. 상위 후보 정렬 품질을 정량화.
  • Hit@K: 상위 K 안에 “관심 후보(활성/상위 타깃)”가 들어갔는지 비율로 평가.
  • 값 범위는 -1 ~ 1:
    • +1: 순위 완전 일치 (가장 이상적)
    • 0 부근: 순위 관계 거의 없음
    • -1: 순위 완전 반대
  • 실무 해석(경험적 가이드):
    • 0.00 ~ 0.10: 거의 무관
    • 0.10 ~ 0.30: 약한 상관
    • 0.30 ~ 0.50: 중간 상관
    • 0.50 ~ 0.70: 강한 상관
    • 0.70+: 매우 강한 상관
  • 왜 Spearman이 중요한가? 약물 스크리닝/랭킹 문제에서는 “절대값 예측 오차”뿐 아니라 우선순위를 맞추는 능력이 핵심이기 때문. Spearman이 높으면 Top 후보 선별의 신뢰도가 올라간다.
  • 이번 결과에서는 baseline 약 0.25(약한 상관)에서 new-C 약 0.48(중간 상관)로 올라, 랭킹 품질이 의미 있게 개선된 것으로 해석 가능.

주석 보기: FE 파이프라인에서 무엇을 어떤 순서로 하나 (`nextflow/scripts/build_features.py`)
  • 병합: 라벨 ↔ sample 피처(inner) ↔ drug 피처(left), 키는 sample_id, canonical_drug_id.
  • 누수 제거: --leakage-cols에 지정된 열 드롭(배치별 설정).
  • 고결측 컬럼 제거: 보호열(key·라벨) 제외, 열 단위 결측 비율 > missing_threshold이면 컬럼 전체 삭제.
  • 결측 대체(impute): 수치형 → 열 중앙값(median), 비수치형 → 문자열 "UNK".
  • 저분산 제거: variance_threshold 기본 0.0 → 수치형에서 분산 ≤ 0인 열 제거(상수열·정보량 없음).
  • 산출: features.parquet, labels.parquet, feature_manifest.json(제거된 컬럼·임pute 방식 등 메타).
  • DL 브랜치(옵션): 동일 wide 표에 대해 수치 Z-score 표준화한 features_dl.parquet — 이번 SageMaker tree 파일럿은 tree 산출물 기준.
주석 보기: 결측·이상치·“이상값”을 어떻게 정의했나
  • 결측 처리: (1) 임계 초과 열은 통째로 버림 (2) 남은 셀은 중앙값/UNK로 채움. “행 단위로 결측 많으면 제거”가 아니라 열 단위 high-missing drop이 핵심.
  • 이상치(outlier) 전용 클리핑: 현재 build_features.py에는 IQR/clip 같은 전역 이상값 제거 단계 없음.
  • 대신 쓰는 “품질 필터”: 저분산 컬럼 제거 = 변화가 없는 피처 제거. 이는 통계적 outlier 탐지와는 다른 목적(모델 안정·불필요 차원 감소).
  • 학습 단계 추가 규칙: train_tabular.py는 FE 이후에도 dtype이 numeric인 열만 사용하고, 남은 수치 결측을 0.0으로 한 번 더 채움(문자 SMILES 등은 이 단계에서 제외 → C new에서는 해시로 별도 결합).
  • wide 17,922 vs 학습 17,917: parquet 전체 열 중 일부는 비수치(문자열 SMILES 등)라 tree 학습에서 빠져 n_features_numeric ≈ 17,917로 집계됨.
