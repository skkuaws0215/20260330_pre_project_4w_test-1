4) 지표 정의 / 실험 해석 기준

주석 보기: 각 지표의 의미
  • RMSE: Root Mean Squared Error, 큰 오차(outlier)에 더 민감한 오차 지표. 낮을수록 좋다.
  • MAE: Mean Absolute Error, 평균 절대 오차. 평균적으로 얼마나 틀리는지 보는 지표. 낮을수록 좋다.
  • Spearman: 순위 상관계수. 정답 순위와 예측 순위가 얼마나 일치하는지(단조 관계)를 본다. 높을수록 좋다.
  • NDCG@20: 상위 20개 랭킹 품질 지표. 중요한 정답을 상위에 올려둘수록 값이 커진다. 높을수록 좋다.
  • Hit@20: 상위 20개 안에 좋은 후보가 포함될 확률(비율). 높을수록 좋다.
  • Delta: 전 단계 대비 변화량. 본 대시보드에서는 Spearman/NDCG 기준으로 +가 개선.
  • 붉은색(bold): 각 비교에서 best 또는 강조 포인트를 표시한다.

위 설정의 의미: 모델은 XGBoost 회귀, 입력은 pair_features_newfe_v2.parquet, 조건은 LINCS feature 제외 + sample_id, canonical_drug_id 기준 labels inner join, seed는 42로 고정했다. CV는 5-fold KFold(shuffle=True, random_state=42)로 fold별 RMSE/MAE/Spearman/NDCG@20을 계산했고, 튜닝은 지정된 탐색공간에서 랜덤 20 trial을 돌려 Spearman (우선) / NDCG@20 (보조) 기준으로 best 조합을 선택했다.

주석 보기: 파라미터/숫자 의미 풀이
  • max_depth: 트리 최대 깊이. 값이 클수록 복잡한 패턴을 학습하지만 과적합 위험도 커진다. 이번 best는 4(상대적으로 얕은 트리).
  • lr (learning_rate): 학습률. 한 번에 얼마나 크게 업데이트할지 정한다. 작을수록 안정적이지만 많은 트리가 필요하다. best는 0.05.
  • n_estimators: 트리 개수. 많을수록 표현력은 늘지만 시간/과적합 부담이 커진다. best는 400.
  • subsample: 각 트리를 학습할 때 사용하는 행 샘플 비율. 0.8은 매 트리마다 데이터 80%만 사용해 분산/과적합을 줄이는 설정.
  • colsample (colsample_bytree): 각 트리에서 사용하는 피처 비율. 0.8은 매 트리마다 피처 80%만 사용해 일반화를 돕는 설정.
  • 5-fold KFold: 데이터를 5개 조각으로 나눠 4개로 학습+1개로 검증을 5번 반복. 한 번의 split 우연성보다 안정적인 평균 성능을 본다.
  • 20 trials 중 최고 조합: 탐색공간에서 20개 파라미터 조합을 시험해, Spearman 평균이 가장 높고(동률 시 NDCG@20 보조) 좋은 조합을 선택했다는 뜻.
  • mean ± std: 5개 fold 성능의 평균과 표준편차. mean이 좋고 std가 작을수록 성능이 높고 fold 간 변동이 적다고 해석한다.

  • 입력: xgb_tuning_results.csv (20 trials).
  • 권장 축: x축 max_depth/n_estimators, y축 cv_spearman_mean, 색상 cv_ndcg20_mean.
  • 해석: 고성능 점이 특정 구간에 모이면 그 파라미터 영역을 다음 탐색의 중심으로 설정.
  • 현재 상태: trial 결과 확보 완료, scatter/heatmap은 후속 렌더링 단계에서 추가.
학습곡선 (learning curve): 데이터량/트리 수에 따른 추세
  • 목적: 데이터량 또는 트리 수 증가 시 성능 개선이 포화되는지 확인.
  • 권장 구성: train fraction(20/40/60/80/100%) 또는 n_estimators 단계별 CV 성능 곡선.
  • 해석: train/valid 간격이 크면 과적합, 둘 다 낮으면 언더피팅 가능성. 포화 지점 이후는 비용 대비 이득이 작음.
  • 현재 상태: 본 런은 고정 데이터량 CV+튜닝 중심이라 learning curve는 후속 실험으로 분리.
잔차 플롯 (residual): 오차 편향/이상치 확인
  • 목적: 예측값 구간별로 오차가 체계적으로 치우치는지(편향) 확인.
  • 권장 플롯: y_pred vs residual, 또는 y_true vs residual 산점도.
  • 해석: 잔차가 0 주변에 랜덤하게 퍼지면 양호. 특정 구간에서 한쪽으로 치우치면 보정 필요.
  • 현재 상태: CV fold 예측 저장을 추가하면 즉시 생성 가능(현재는 metric 집계 중심).
Feature importance/bar plot: 기여 피처 확인
  • 목적: 어떤 피처 그룹(예: target, chem)이 성능 개선에 기여했는지 설명 가능성 확보.
  • 권장 산출: XGBoost gain 기반 top-N bar plot + 그룹별 누적 기여율.
  • 해석: 상위 피처가 한 그룹에 과도하게 몰리면 데이터 의존 편향 가능성을 점검.
  • 현재 상태: best params 확정됨. 최종 학습 모델에서 importance 추출해 시각화 가능.
SHAP 숫자(예: 0.102, 0.092)가 의미하는 것
  • 값 정의: 각 숫자는 mean(|SHAP value|)로, 해당 피처가 예측값을 평균적으로 얼마나 크게 움직였는지(절대 영향 크기)를 뜻한다.
  • 단위: 타깃(label_regression)과 같은 단위. 예: 0.102는 평균적으로 예측값을 약 0.102 규모로 이동시켰다는 의미.
  • 비교 해석: 값이 클수록 영향력이 큼. 예: 0.102는 0.052보다 약 2배 큰 영향.
  • 주의점: SHAP는 영향 크기 해석 도구이지 인과관계 증명이 아님. 현재 값은 샘플 2000개 기준이라 샘플링에 따라 소폭 변할 수 있음.
