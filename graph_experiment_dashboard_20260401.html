Graph Experiment Dashboard

Graph 군은 GNN(학습 기반)만이 아니라, PPI/유전자 네트워크 상의 거리·근접도(규칙 기반) 접근도 포함합니다. 로컬 검증 완료: 최종 대표는 drug-group CV Spearman mean 기준 GCN. Round1 행 단위 CV에서는 GraphSAGE가 Spearman 상위였으나 temporary candidate로만 기록 — 최종 확정은 group CV까지 본 뒤 GCN으로 정리. GCN 경량 튜닝(A–D, 동일 drug-group CV)을 수행했으나 1차 지표 Spearman mean 기준 의미 있는 개선(베이스라인 A 대비 ≥0.01)은 없음 → 운영·SageMaker용 GCN 하이퍼파라미터는 기존 baseline(A: hidden 64, lr 1e-3, weight_decay 1e-5) 유지. 다음 단계는 SageMaker에서 GCN 최종 실행(DL 대시보드 §0과 함께 3종).

4) Graph 군 baseline (초기 3종, 로컬)

초기 단계는 아래 3개만 실행합니다. 동일 5-fold에서 graph_family_comparison.csv에 한 줄씩 기록합니다.

#방법역할핵심상태
1 Network Proximity rule-based baseline drug-target vs disease gene 네트워크 거리/근접도, 무작위 대조 대비 z-score; 학습 없음. 샘플별이 아니라 약물 단위 스코어를 모든 pair 행에 브로드캐스트 (rule-based, non-sample-specific graph baseline). Round1 실행됨
2 GraphSAGE GNN primary baseline sample-aggregate; drug-gene graph + sample-side feature 결합 구조 Round1 실행됨 · Round1에서 Spearman 기준 temporary graph representative candidate (행 단위 CV·공유 drug node로 optimistic bias 가능 → group CV로 확정 보류)
3 GCN Graph 군 확정 대표 (SageMaker) 기본 message passing; drug-group CV에서 Spearman mean 최고 → 최종 대표. GraphSAGE는 Round1 row-CV temporary만. 미니튜닝 후에도 설정은 baseline(A) 유지(§8 표 참고). Round1 + group CV + GCN 튜닝(A–D) 완료 · 대표 확정·hparams baseline

주의: 초기에는 GAT / R-GCN(Hetero) 제외. GraphSAGE·GCN 결과 확인 후 필요 시 확장.

주석: GNN vs Network Proximity 비교 목적
  • GNN은 임베딩·메시지 패싱으로 패턴을 학습합니다.
  • Network Proximity는 “타깃–질병 유전자가 그래프에서 얼마나 가깝냐”는 해석 가능한 규칙만으로 스코어를 만듭니다.
  • 동일 fold에서 둘 다 평가하면, 그래프 정보가 학습 없이도 얼마나 신호를 주는지 vs 학습으로 얼마나 끌어올리는지 구분할 수 있습니다.

8) Drug-group CV (stricter validation)

목적: 행 단위 KFold + GNN의 공유 drug node 표현 때문에 생길 수 있는 optimistic bias를 줄이기 위해, canonical_drug_id 기준으로 train·valid를 분리한 뒤 동일 3종을 재평가합니다. Network Proximity는 폴드별 train에서 y ~ a·z+b 보정 후 RMSE/MAE를 계산하며, 비정상 보정 시 train 평균 예측으로 폴백합니다.

요약

  • Proximity / GraphSAGE / GCN 행의 RMSE·MAE·Spearman·NDCG@20·Hit@20은 검증 폴드에서 산출한 뒤 5폴드 산술평균으로 요약됩니다.
  • Graph 군 내 모델 간 1차 비교는 Spearman mean (merge_graph_family_outputs의 selection_rule과 동일).
  • Network Proximity의 RMSE/MAE는 train 폴드에서 보정 후 valid에 적용; 비정상일 때 train 평균 예측 폴백 (graph_schema_groupcv 등 참고).
  • Spearman·NDCG·Hit은 Proximity도 원시 z 기반으로 계산될 수 있음 — 구현·스키마는 graph_schema*.json, run_network_proximity_baseline.py 참고.
  • GCN 대표 확정: 본 §8의 3종 표에서 Spearman mean 최대. GraphSAGE는 Round1 row CV에서만 temporary candidate.
  • GCN 하이퍼파라미터: 동일 drug-group CV 미니튜닝(A–D) 후에도 baseline A 대비 Spearman mean ≥0.01 개선이 없어 hidden 64 · lr 1e-3 · weight_decay 1e-5 유지.

최종 Graph 군 대표: GCN (3종 표에서 Spearman mean 최대). 하이퍼파라미터: 미니튜닝 결과에 따라 baseline 구성 A 유지(위 튜닝 표 참고). 정책 요약: Round1 행 단위 CV에서는 GraphSAGE가 Spearman 기준 잠정 1위였으나 optimistic bias 가능 → GraphSAGE는 temporary candidate로만 JSON·문서에 남기고, 운영·SageMaker 라인업의 Graph 슬롯은 GCN으로 확정. JSON 필드 temporary_graph_representative_candidate는 그 이력용.
