1) 목적

  • graph 기반 접근이 기존 ML(XGBoost) / DL(ResidualMLP) 대비 의미 있는 성능 개선이 있는지 확인
  • Graph 군에서 대표 모델 1종 선정 후 최종 챔피언 비교군에 포함
  • 규칙 기반(Network Proximity)과 학습 기반(GNN: GraphSAGE, GCN)을 동일 데이터·동일 split·동일 지표로 비교
  • 성능뿐 아니라 안정성(std), 운영 복잡도까지 함께 판단

4) Graph 군 baseline (초기 3종, 로컬)

초기 단계는 아래 3개만 실행합니다. 동일 5-fold에서 graph_family_comparison.csv에 한 줄씩 기록합니다.

#방법역할핵심상태
1 Network Proximity rule-based baseline drug-target vs disease gene 네트워크 거리/근접도, 무작위 대조 대비 z-score; 학습 없음. 샘플별이 아니라 약물 단위 스코어를 모든 pair 행에 브로드캐스트 (rule-based, non-sample-specific graph baseline). Round1 실행됨
2 GraphSAGE GNN primary baseline sample-aggregate; drug-gene graph + sample-side feature 결합 구조 Round1 실행됨
3 GCN GNN secondary baseline 기본 message passing; GraphSAGE와 동일 그래프/스플릿으로 비교 Round1 실행됨

주의: 초기에는 GAT / R-GCN(Hetero) 제외. GraphSAGE·GCN 결과 확인 후 필요 시 확장.

주석: GNN vs Network Proximity 비교 목적
  • GNN은 임베딩·메시지 패싱으로 패턴을 학습합니다.
  • Network Proximity는 “타깃–질병 유전자가 그래프에서 얼마나 가깝냐”는 해석 가능한 규칙만으로 스코어를 만듭니다.
  • 동일 fold에서 둘 다 평가하면, 그래프 정보가 학습 없이도 얼마나 신호를 주는지 vs 학습으로 얼마나 끌어올리는지 구분할 수 있습니다.

5) 평가 지표 정의

표기정의해석
SpSpearman rank correlation클수록 좋음
RMSE회귀 오차 제곱평균제곱근작을수록 좋음
NDCG@20상위 20개 랭킹 품질클수록 좋음
Hit@20상위 20개 내 hit 비율클수록 좋음
stdfold 간 Spearman 표준편차작을수록 안정적
MAE평균 절대 오차작을수록 좋음
