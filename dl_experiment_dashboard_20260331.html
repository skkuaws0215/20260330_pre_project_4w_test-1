주석: 표의「구조/핵심 아이디어」열 설명
  • XGBoost — Tree 기반 best baseline: 그래디언트 부스팅 결정트리 앙상블. 피처 스케일에 덜 민감하고 비선형·상호작용을 자동으로 나눠 학습합니다. DL과 비교할 때 tabular 강한 전통 ML 기준선 역할입니다.
  • FlatMLP — 기본 MLP baseline: 모든 입력 피처를 한 벡터로 이어 붙인 뒤, 완전연결층만 쌓은 다층 퍼셉트론. 구조가 단순해 DL의 출발점으로 두고, 구조 개선형 MLP와의 차이를 보기 위한 대조군입니다.
  • VAE — Encoder/Decoder + latent regression: 입력을 저차원 잠복공간(latent)으로 압축·복원하는 변분 오토인코더에, 잠복 벡터로부터 타깃을 예측하는 회귀 헤드를 붙인 형태. 재구성·KL·예측 손실을 함께 쓰며 “표현 학습 + 회귀”를 동시에 시도합니다.
  • TabNet — Attentive tabular DL: 단계마다 어떤 피처에 집중할지 학습하는 attention 기반 tabular 전용 신경망. 해석 가능성을 노리지만, 데이터·설정에 따라 baseline 대비 이득이 없을 수 있습니다.
  • BlockWiseMLP — feature block encoder + concat head: pathway / chemistry / target 등 의미별로 피처를 나눠 각 블록에 작은 인코더를 두고, 블록 임베딩을 이어 붙인 뒤 최종 회귀 헤드로 예측합니다. 도메인 구조를 네트워크에 반영하려는 설계입니다. (현재 데이터에서 pathway 블록이 비면 chem·target 분리 수준만 의미 있습니다.)
  • ResidualMLP — flat MLP + residual blocks: 깊은 완전연결 경로에서 skip connection으로 그래디언트·학습 안정성을 보완한 MLP. 구조만 바꿔 flat 대비 성능 개선이 있는지 보기 위한 변형입니다.
주석: 평가 지표(RMSE, MAE, Spearman, NDCG@20, Hit@20) — 기호·식·필요 조건

공통 기호

  • i: 검증(holdout 또는 CV의 validation fold)에 포함된 한 행(한 drug–sample 쌍) 인덱스. 집합 크기를 N이라 하면 i = 1, …, N.
  • xᵢ: i번째 행의 입력 피처 벡터(join·전처리 후 수치 컬럼 전체). 회귀 식에는 보통 나타나지 않고, “모델 입력”을 가리킬 때 씁니다.
  • yᵢ: i번째 행의 실제 타깃(본 실험에서는 label_regression). 관측값·정답.
  • ŷᵢ (y-hat): 동일한 xᵢ에 대해 모델이 낸 예측 회귀값. 스크립트에서는 검증 구간에 대해 한 번에 벡터 y_true, y_pred로 계산합니다.

RMSE (Root Mean Squared Error)

  • 식: RMSE = √( (1/N) · Σᵢ (yᵢ − ŷᵢ)² )
  • 해석: 오차 제곱 평균의 제곱근이라 큰 오차에 더 민감. 단위는 타깃과 동일.
  • 필요 조건: 모든 i에 대해 yᵢ, ŷᵢ가 실수로 정의되어 있고 결측이 없을 것(본 파이프라인에서는 결측 0 처리 후 사용).

MAE (Mean Absolute Error)

  • 식: MAE = (1/N) · Σᵢ |yᵢ − ŷᵢ|
  • 해석: 절대오차 평균. 이상치에 RMSE보다 덜 민감한 편.
  • 필요 조건: RMSE와 동일하게 쌍 (yᵢ, ŷᵢ)이 N개 필요.

Spearman (Spearman rank correlation)

  • 식(개념): 벡터 y와 ŷ를 각각 순위로 바꾼 뒤, 그 두 순위열 간의 피어슨 상관과 동치인 순위 상관계수.
  • 해석: 선형이 아니라 단조(monotonic) 관계가 맞는지에 가깝게 반영. 스케일 이동에는 불변에 가깝습니다.
  • 필요 조건: y 또는 ŷ가 상수(분산 0)이면 정의가 불안정해 NaN이 될 수 있음. N ≥ 2.

NDCG@20 (Normalized Discounted Cumulative Gain, 컷오프 20)

  • 단위: 동일 sample_id 아래에 여러 canonical_drug_id(약물 후보)가 있을 때, 한 환자(샘플) 안에서의 랭킹 품질을 봅니다.
  • 관련도(relevance): 구현에서는 그룹 내 yᵢ(실제 반응 강도)를 relevance로 쓰고, ŷᵢ로 약물을 정렬해 랭킹을 만듭니다. sklearn ndcg_score는 음수 relevance를 허용하지 않아, 그룹 내 최솟값이 음수이면 전체 y를 그룹별로 평행 이동해 0 이상으로 맞춘 뒤 계산합니다.
  • 식(개념): 상위 K(여기서 K=min(20, 그룹 크기))에 대해, 할인 누적 이득을 이상적 랭킹 대비로 정규화한 값. 1에 가까울수록 좋은 랭킹.
  • 필요 조건: 해당 sample_id 그룹에 서로 다른 약물이 2개 이상인 행만 포함(크기 1 그룹은 스킵 후, 남은 그룹들에 대해 평균).

Hit@20

  • 정의(본 저장소 구현): sample_id별로, y가 큰 순서 상위 K개 약물 인덱스 집합과 ŷ가 큰 순서 상위 K개 집합(K=min(20, 그룹 크기))의 교집합이 비어 있지 않으면 1, 아니면 0으로 두고, 그룹별로 구한 0/1을 평균합니다.
  • 해석: “진짜 좋은 약이 예측 상위권에 하나라도 걸렸는가”에 가까운 완화된 히트율.
  • 필요 조건: NDCG와 같이 그룹당 최소 2행. 그룹이 없으면 지표는 정의되지 않을 수 있습니다.

정리: RMSE·MAE·Spearman은 전체 검증 행 N개에 대한 쌍wise (yᵢ, ŷᵢ) 지표이고, NDCG@20·Hit@20은 sample_id별 리스트 랭킹 지표라서, “같은 숫자인데 의미가 다름”을 구분해 읽어야 합니다.

입력 · pair_features_newfe_v2.parquet (LINCS 제외)가 의미하는 것

  • 무엇의 테이블인가: 새 FE 배치(예: 20260331 파이프라인 산출)에서 만든 쌍 단위(sample–drug) 피처 테이블의 한 변종입니다. v2는 target 관련 피처가 반영된(target-only 강화) 버전으로, newfe 대비 target 신호가 더 들어간 구성입니다.
  • LINCS 제외: 학습 시 컬럼명에 lincs가 들어간 피처는 사용하지 않습니다. BRD–내부 약물 매핑이 확정되기 전 단계라 LINCS 브리지를 쓰지 않기로 한 정책과 맞춥니다.
  • 실제로 붙는 데이터: 실행 스크립트에서는 labels(타깃)와 sample_id, canonical_drug_id로 inner join한 뒤, 위 key·타깃을 제외한 나머지 수치 컬럼이 입력 피처가 됩니다. 대략적으로 chem 계열(지문·서술子等), target_ 접두 타깃/경로 요약 피처, 필요 시 pathway 라벨이 들어간 컬럼 등이 섞입니다. (현재 스키마에서는 pathway 블록이 비어 있을 수 있음 — Block-wise 주석 참고.)
  • 비교 공정성: 모든 DL 행이 동일 파일을 가리키는 이유는, 동일한 피처 공간·동일 join/split 정책 아래에서 “모델 구조”만 바꿔 보기 위해서입니다.

모델별 핵심 설정 — 왜 이렇게 잡았는지

  • FlatMLP — 2~3 hidden layer baseline: 가장 단순한 fully-connected 스택입니다. 과도한 깊이·폭을 피해 “DL 기준선”만 잡고, 이후 Residual/Block-wise와의 구조 효과만 분리해서 보기 위한 설정입니다. 별도 대규모 HPO는 하지 않았습니다.
  • VAE — latent dim / beta / recon + pred loss: 표현을 latent로 압축하면서 회귀 타깃까지 맞추는 VAE+회귀 헤드 표준 형태입니다. latent 차원·KL(β)은 소수의 후보만 두고 Spearman 등으로 대표 설정을 고른 수준으로, 생성 품질까지 최적화하는 단계는 아닙니다.
  • TabNet — n_d=16, n_a=16, n_steps=4 baseline: 논문·권장 초기값에 가까운 가벼운 baseline입니다. 탭넷은 튜닝에 민감하고 학습 비용이 있어, 이번 단계에서는 “구조가 이 데이터에서 이득인가”만 빠르게 점검하는 목적에 맞췄습니다.
  • Block-wise MLP — pathway/chem/target 블록 + concat: 도메인상 서로 다른 성격의 피처를 별도 인코더로 처리한 뒤 합치려는 설계입니다. pathway 컬럼이 풍부해질수록 의미가 커지고, 없을 때는 사실상 chem vs target 분리 수준의 구조 비교가 됩니다.
  • Residual MLP — residual block(3): 깊이를 약간 준 MLP에서 skip connection으로 학습 붕괴·포화를 완화하고, flat 대비 표현력·안정성만 바꾼 변형입니다. 블록 개수는 “baseline 수준”으로 3을 고정해 두었습니다.

공통: 이번 DL 군 실험은 최적 하이퍼파라미터 탐색보다 구조·가족 간 상대 비교를 우선합니다. 최종 챔피언이 정해진 뒤 SageMaker 등에서 해당 모델만 집중 튜닝하는 편이 효율적입니다.

주석: Holdout / CV / XGBoost_tuned 의미 · 붉은색 하이라이트 규칙

Holdout

  • 전체 데이터를 한 번만 train / validation(또는 test)으로 나눈 뒤, 고정된 검증 구간 한 덩어리에서만 지표를 계산한 결과입니다.
  • 여기서는 train_test_split(test_size=0.2, random_state=42) 등으로 만든 단일 검증 분할에 해당합니다.
  • 장점: 해석이 단순하고 보고가 쉬움. 단점: 분할 운에 따라 수치가 들쭉날쭉할 수 있어, CV와 함께 보는 것이 안전합니다.

5-fold CV (cross-validation)

  • 데이터를 5개 폴드로 나누어, 각 폴드를 한 번씩 검증으로 쓰고 나머지로 학습하는 과정을 5번 반복합니다.
  • 표의 mean±std는 5번의 검증 지표 평균과 표준편차로, holdout 한 번보다 분할 운에 대한 안정성을 보기 좋습니다.
  • 이 표의 CV 블록은 XGBoost(tuned) vs ResidualMLP만 포함합니다(동일 폴드 비교용).

XGBoost_tuned

  • 고정 “기본” 하이퍼파라미터가 아니라, 로컬에서 CV 기반 간단 탐색(예: randomized/grid, 최대 20 trial)으로 고른 조합으로 학습한 XGBoost를 뜻합니다.
  • 본 저장소에서는 xgb_cv_tuning/xgb_best_params.json에 기록된 max_depth, learning_rate, n_estimators 등이 그 결과입니다.
  • Holdout 행의 XGBoost는 동일 데이터셋 비교 맥락의 트리 기준이며, 이름에 tuned가 없어도 실험 시점별로 기본/튜닝 여부가 다를 수 있으니, CV 행은 명시적으로 tuned CV 스펙입니다.

붉은색(.best) 표시 규칙

  • 같은 experiment 열 값끼리만 비교합니다. 즉 Holdout 6행 안에서 열별 최선, 5-fold CV 2행 안에서 열별 최선입니다. Holdout과 CV 숫자는 직접 섞어 승패를 말하지 않습니다.
  • RMSE, MAE: 더 작을수록 좋음 → 해당 그룹에서 최소인 셀만 표시.
  • Spearman, NDCG@20, Hit@20: 더 클수록 좋음 → 해당 그룹에서 최대인 셀만 표시.
  • 동률이면 동률인 셀 모두 붉은색으로 표시합니다(예: Holdout에서 MAE 1.583 공동 1위, Hit@20 전원 1.000).

요약: 단일 holdout과 5-fold CV 모두에서 ResidualMLP가 안정적으로 상위권입니다. DL 군 전용 동일 split 비교는 아래 6) 표를 기준으로 합니다.

8) 의사결정 참고 — Agent 요약 의견

아래는 수치·표(0), 5)~7))를 바탕으로 한 비공식 의견입니다. 최종 판단은 팀 목표·리스크·운영(배포·설명 가능성)에 맞게 조정하세요.

요약 (업데이트): 로컬 단계에서는 pathway 미포함·포함을 분리해 본 것이 여전히 유효합니다(§6 GMT 주석, §7). SageMaker 최종 실행 3종은 §0과 같이 ML: XGBoost · DL: ResidualMLP · Graph: GCN으로 확정합니다. BlockWiseMLP는 pathway 풍부 시 재평가 가치가 있으나, 이번 최종 3-track 라인업에서는 제외하고 필요 시 후속 실험으로 둡니다.

주석: DL 군 채택·제외·ML/Graph와의 관계

DL 군으로 쓸 만한 것

  • ResidualMLP: pathway 미포함 비교(6, 7)에서 상위권을 안정적으로 유지. 운영 단순성 측면에서 기본 대표 후보로 적합.
  • BlockWiseMLP: pathway=0일 때는 chem/target 분리 수준이지만, pathway=50 재평가에서 성능이 크게 올라 공동 상위 후보 근거 확보.
  • FlatMLP: 베이스라인으로는 유효하나, Residual이 dominate하면 군 대표 라인업에서 빼도 무방.

빼는 편이 나은 것 (현 증거·현 목표 기준)

  • TabNet: 동일 조건에서 격차가 커서 DL 군 대표 후보로는 비추천. 별도 튜닝·피처 개편 전까지 의사결정 코어에서 제외해도 됨.
  • VAE: NDCG는 장점이 있을 수 있으나 Spearman(순위)이 약함. “순위/랭킹”이 주지표면 DL 대표 경쟁에서 후순위. 생성·표현 학습을 목적으로 할 때만 별도 검토.

ML·Graph와의 관계

  • ML 군: 정렬·해석·운운 측면에서 XGBoost가 대표 후보로 자연스러움. DL 결과가 우세하다 해도 트리 기반은 벤치마크·SHAP 등으로 병행 가치가 큼.
  • Graph 군: 로컬에서 Proximity / GraphSAGE / GCN 완료. Graph representative: GCN. 미니튜닝(A–D) 완료, 내부 best D·+0.0054 < 0.01 → baseline A 유지, E 보류. GraphSAGE는 Round1 row-CV temporary candidate만.
  • 의사결정 순서 (현재): 군별 대표 확정(§0) → SageMaker에서 3종 최종 학습·지표 정리 → 필요 시 동일 holdout/정책으로 챔피언 비교.
