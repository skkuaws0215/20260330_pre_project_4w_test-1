주석: 표의「구조/핵심 아이디어」열 설명
  • XGBoost — Tree 기반 best baseline: 그래디언트 부스팅 결정트리 앙상블. 피처 스케일에 덜 민감하고 비선형·상호작용을 자동으로 나눠 학습합니다. DL과 비교할 때 tabular 강한 전통 ML 기준선 역할입니다.
  • FlatMLP — 기본 MLP baseline: 모든 입력 피처를 한 벡터로 이어 붙인 뒤, 완전연결층만 쌓은 다층 퍼셉트론. 구조가 단순해 DL의 출발점으로 두고, 구조 개선형 MLP와의 차이를 보기 위한 대조군입니다.
  • VAE — Encoder/Decoder + latent regression: 입력을 저차원 잠복공간(latent)으로 압축·복원하는 변분 오토인코더에, 잠복 벡터로부터 타깃을 예측하는 회귀 헤드를 붙인 형태. 재구성·KL·예측 손실을 함께 쓰며 “표현 학습 + 회귀”를 동시에 시도합니다.
  • TabNet — Attentive tabular DL: 단계마다 어떤 피처에 집중할지 학습하는 attention 기반 tabular 전용 신경망. 해석 가능성을 노리지만, 데이터·설정에 따라 baseline 대비 이득이 없을 수 있습니다.
  • BlockWiseMLP — feature block encoder + concat head: pathway / chemistry / target 등 의미별로 피처를 나눠 각 블록에 작은 인코더를 두고, 블록 임베딩을 이어 붙인 뒤 최종 회귀 헤드로 예측합니다. 도메인 구조를 네트워크에 반영하려는 설계입니다. (현재 데이터에서 pathway 블록이 비면 chem·target 분리 수준만 의미 있습니다.)
  • ResidualMLP — flat MLP + residual blocks: 깊은 완전연결 경로에서 skip connection으로 그래디언트·학습 안정성을 보완한 MLP. 구조만 바꿔 flat 대비 성능 개선이 있는지 보기 위한 변형입니다.

이번 섹션은 현재까지 사용된 모든 DL/비교 모델의 실행/평가 상태를 통합 요약합니다.

입력 · pair_features_newfe_v2.parquet (LINCS 제외)가 의미하는 것

  • 무엇의 테이블인가: 새 FE 배치(예: 20260331 파이프라인 산출)에서 만든 쌍 단위(sample–drug) 피처 테이블의 한 변종입니다. v2는 target 관련 피처가 반영된(target-only 강화) 버전으로, newfe 대비 target 신호가 더 들어간 구성입니다.
  • LINCS 제외: 학습 시 컬럼명에 lincs가 들어간 피처는 사용하지 않습니다. BRD–내부 약물 매핑이 확정되기 전 단계라 LINCS 브리지를 쓰지 않기로 한 정책과 맞춥니다.
  • 실제로 붙는 데이터: 실행 스크립트에서는 labels(타깃)와 sample_id, canonical_drug_id로 inner join한 뒤, 위 key·타깃을 제외한 나머지 수치 컬럼이 입력 피처가 됩니다. 대략적으로 chem 계열(지문·서술子等), target_ 접두 타깃/경로 요약 피처, 필요 시 pathway 라벨이 들어간 컬럼 등이 섞입니다. (현재 스키마에서는 pathway 블록이 비어 있을 수 있음 — Block-wise 주석 참고.)
  • 비교 공정성: 모든 DL 행이 동일 파일을 가리키는 이유는, 동일한 피처 공간·동일 join/split 정책 아래에서 “모델 구조”만 바꿔 보기 위해서입니다.

모델별 핵심 설정 — 왜 이렇게 잡았는지

  • FlatMLP — 2~3 hidden layer baseline: 가장 단순한 fully-connected 스택입니다. 과도한 깊이·폭을 피해 “DL 기준선”만 잡고, 이후 Residual/Block-wise와의 구조 효과만 분리해서 보기 위한 설정입니다. 별도 대규모 HPO는 하지 않았습니다.
  • VAE — latent dim / beta / recon + pred loss: 표현을 latent로 압축하면서 회귀 타깃까지 맞추는 VAE+회귀 헤드 표준 형태입니다. latent 차원·KL(β)은 소수의 후보만 두고 Spearman 등으로 대표 설정을 고른 수준으로, 생성 품질까지 최적화하는 단계는 아닙니다.
  • TabNet — n_d=16, n_a=16, n_steps=4 baseline: 논문·권장 초기값에 가까운 가벼운 baseline입니다. 탭넷은 튜닝에 민감하고 학습 비용이 있어, 이번 단계에서는 “구조가 이 데이터에서 이득인가”만 빠르게 점검하는 목적에 맞췄습니다.
  • Block-wise MLP — pathway/chem/target 블록 + concat: 도메인상 서로 다른 성격의 피처를 별도 인코더로 처리한 뒤 합치려는 설계입니다. pathway 컬럼이 풍부해질수록 의미가 커지고, 없을 때는 사실상 chem vs target 분리 수준의 구조 비교가 됩니다.
  • Residual MLP — residual block(3): 깊이를 약간 준 MLP에서 skip connection으로 학습 붕괴·포화를 완화하고, flat 대비 표현력·안정성만 바꾼 변형입니다. 블록 개수는 “baseline 수준”으로 3을 고정해 두었습니다.

공통: 이번 DL 군 실험은 최적 하이퍼파라미터 탐색보다 구조·가족 간 상대 비교를 우선합니다. 최종 챔피언이 정해진 뒤 SageMaker 등에서 해당 모델만 집중 튜닝하는 편이 효율적입니다.

8) 의사결정 참고 — Agent 요약 의견

아래는 수치·표(5)~7))를 바탕으로 한 비공식 의견입니다. 최종 판단은 팀 목표·리스크·운영(배포·설명 가능성)에 맞게 조정하세요.

요약: DL 군 대표는 ResidualMLP를 1순위로 두는 것이 타당합니다. 동일 holdout·5-fold CV 모두에서 Spearman·오차 지표가 우수하고, 구조가 단순해 재현·튜닝·이관에 유리합니다. BlockWiseMLP는 2순위 후보로 가능하나, 현재 데이터에서 pathway 블록이 비어 있어 실질적으로 chem/target 분리만 의미가 있으므로, pathway 피처 보강 후 재평가를 권장합니다. FlatMLP는 residual이 같은 전제에서 더 나으므로 군 대표를 하나만 둘 경우 생략 가능합니다. 최종 챔피언은 ML 대표 vs DL 대표 vs Graph 대표 단계에서 다시 비교하는 전제가 자연스럽습니다.

주석: DL 군 채택·제외·ML/Graph와의 관계

DL 군으로 쓸 만한 것

  • ResidualMLP: holdout(6)) 및 XGB+MLP 3종 5-fold CV(7)) 모두에서 상위. DL 군 1순위 대표 후보.
  • BlockWiseMLP: Residual에 근접한 2순위. 다만 pathway=0이면 “블록 설계” 검증은 불완전 — 피처 세트 개선 후 재측정 권장.
  • FlatMLP: 베이스라인으로는 유효하나, Residual이 dominate하면 군 대표 라인업에서 빼도 무방.

빼는 편이 나은 것 (현 증거·현 목표 기준)

  • TabNet: 동일 조건에서 격차가 커서 DL 군 대표 후보로는 비추천. 별도 튜닝·피처 개편 전까지 의사결정 코어에서 제외해도 됨.
  • VAE: NDCG는 장점이 있을 수 있으나 Spearman(순위)이 약함. “순위/랭킹”이 주지표면 DL 대표 경쟁에서 후순위. 생성·표현 학습을 목적으로 할 때만 별도 검토.

ML·Graph와의 관계

  • ML 군: 정렬·해석·운운 측면에서 XGBoost가 대표 후보로 자연스러움. DL 결과가 우세하다 해도 트리 기반은 벤치마크·SHAP 등으로 병행 가치가 큼.
  • Graph 군: 아직 미실험. DL 결론은 “tabular DL 범위 내”에 한정되며, Graph 실험 후 세 군 대표를 한 테이블로 묶어 최종 비교하는 흐름이 앞서 합의한 방향과 일치.
  • 의사결정 순서 제안: DL 군 대표(Residual 중심) 확정 → Graph 군 실험·대표 선정 → ML vs DL vs Graph 대표 스펙·지표로 최종 1안(또는 앙상블) 선택.
