AI Drug Discovery pre-project v.1

Streamlit 「AI Drug Discovery — 진행 대시보드」와 같은 요약·플로우 성격을 정적 HTML로 모은 팀4 스냅샷입니다. 세부 표·주석·전체 수치는 아래 링크의 기존 전용 대시보드를 따릅니다. 본 파일은 기존 HTML을 수정하지 않고 신규 추가되었습니다.

스냅샷 · pre-project v.1 결과 중심 요약 Team4 · drug-discovery-joe-raw-data-team4 본선·운영 = LINCS 제외
헷갈림 방지 — 결론: 재-FE 파이프라인에서 LINCS 피처를 만들어 둔 적은 있음(예: pair_lincs_features). 그러나 최종적으로 본선이 맞는 설정은 LINCS 제외이다. ML·DL·SageMaker 학습 입력에서는 lincs_* 컬럼을 쓰지 않았고, METABRIC 공식 외부 랭킹도 no-LINCS 산출물이 기준이다.

Streamlit 요약본: 로컬에서 python -m streamlit run streamlit_app.py (또는 run_dashboard.ps1).

0) 전체 파이프라인 흐름 (단계)

정책 고정: 이 문서에서 말하는 「재-FE로 진행한 본선」은 LINCS 제외가 맞다. (FE 산출물에 LINCS 조각이 있어도 학습·운영 랭킹에는 넣지 않음.)

  1. 팀 공유 전처리 — S3 results/<소스>/ parquet (raw 기반). 팀4는 읽기 전용 입력으로만 사용. ml_ready/는 FE 입력으로 사용하지 않음.
  2. 브리지·1차 FE — prepare_fe_inputs.py 등으로 표준 3종 입력 → Nextflow·Batch로 features.parquet / labels.parquet / manifest (…/features_nextflow_team4/).
  3. 재-FE (fe_re_batch_runs/) — New FE 재생성 시 pathway·Morgan·LINCS(조각 parquet로 존재)·target interaction 등을 설계에 포함할 수 있음. 다만 통합 pair_features_newfe_v2.parquet를 쓸 때도 본선 ML·DL·SageMaker 학습 입력에서는 LINCS 관련 컬럼을 제외하는 정책이며, METABRIC 공식 외부 랭킹은 no-LINCS native 산출물로 고정.
  4. ABC 파일럿 (ML 4모델) — 별도 고정 입력 abc_inputs/20260330_abc_v1/ 에서 A/B/C 비교(SageMaker 4모델). 본선 re-FE와 입력 경로가 다름.
  5. ML·DL·Graph 확장 — re-FE 산출(주로 fe_re_batch_runs/20260331/…/pair_features_newfe_v2.parquet, 학습 시 LINCS 컬럼 제외 정책 등)으로 트리·DL·GNN 탐색 후 군별 대표 선정.
  6. 최종 3종 + SageMaker — XGBoost(tuned) · ResidualMLP · GCN을 클라우드에서 확정 실행 및 듀얼 밸리데이션(CV / holdout).
  7. 앙상블·후처리 — final_ensemble_ranking_v2.csv 등 → METABRIC·ADMET·shortlist 파이프라인.
  8. 외부·약물성 — METABRIC 경로 정정(no-LINCS native 등) + SwissADME(1차 웹 ADME) + ADMETlab 3.0(2차 웹 독성/대사).

1.4 재-FE 루트 — fe_re_batch_runs/ (New FE → 본선 ML·DL·Graph)

결국 맞는 말: 본선·운영은 LINCS 제외. 아래 표의 LINCS는 「파이프라인 중간 산물」일 뿐, 모델 학습·최종 METABRIC 랭킹에는 반영하지 않음.

Early Batch FE(§1.2)·ABC 입력(§1.3) 이후, pair_features_newfe(_v2) 를 다시 만든 실행 묶음입니다. manifest 상 pair_lincs 블록·pair_lincs_features.parquet 등으로 LINCS 피처를 FE 단계에서 생성·병합할 수 있음. 그러나 결론적으로 본선은 LINCS 제외: 트리닝/추론 시 lincs_* 컬럼을 떼고 학습·SageMaker·앙상블에 넣었고, METABRIC 운영 랭킹은 metabric_native_*_no_lincs·검증 리포트로 no-LINCS를 공식 경로로 둠.
